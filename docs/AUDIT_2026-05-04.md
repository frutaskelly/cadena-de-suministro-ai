## 🟠 ALTOS (12)

### Backend
- **H1** N+1 queries en `/remisiones` — falta `selectinload(LineaRemision.producto)` cascada cuando frontend acceda producto. (~500 queries por request en peor caso)
- **H2** `HTTPException(detail=str(e))` expone trazas internas tipo `RemisionError(...)`. Mapear a códigos públicos.
- **H3** Sin response_model en endpoints POST como `create_pedidos_from_batch`. OpenAPI no documenta respuesta.
- **H4** Sin logging estructurado en services/. Excepciones sin contexto (tenant_id, IDs). Debugging post-mortem imposible.
- **H5** `FACTURAMA_API_KEY` y demás envvars críticos sin `field_validator` en pydantic-settings. Fail silente en startup.

### DB / Security
- **H6** RLS policies presentes pero `app_user` rol no-superuser no creado en Supabase — postgres rol bypassa todo. Validar.
- **H7** Soft delete (`deleted_at`) sin trigger ni view: si olvidamos `.filter(deleted_at.is_(None))` en un endpoint, leak de datos eliminados.
- **H8** JSONB sin GIN index (`productos.presentaciones`, `pedidos.ai_warnings`, `*.config`, `*.metadata`). Queries con `@>` full-scan.
- **H9** `csds.password_kms_id` es plaintext. CSDs son datos críticos SAT, deben cifrarse en reposo (KMS).
- **H10** `events_log.before_state/after_state` puede contener RFC sin redactar. PII en audit log.

### Frontend
- **H11** Sin error.tsx ni loading.tsx en ningún route segment. Crashes silenciosos; UI en blanco durante 2-5s en tablas grandes.
- **H12** Tipos `Record<string, unknown>` en `domicilio_fiscal`, `config`, `metadata_doc`, `custom_fields` — runtime errors si backend cambia shape.

## 🟡 MEDIOS (14)

### Backend
- M1 `app.on_event("startup")` deprecado — usar lifespan handlers.
- M2 Endpoints async-marked declaran funciones `def` (no `async def`) — confuso, no aprovecha async.
- M3 Inconsistencia naming `metadata` (column) vs `metadata_doc` (Python attribute) en `DocumentoGenerado`.
- M4 `services/remisiones.py:_move_inventory_remision_out` usa `if hasattr(rem, "lineas")` — defensive coding apunta a relación mal cargada (lazy loading issue).

### Frontend
- M5 12 páginas re-fetching tenant_id sin Context global. Cuando cambia, 1-5s delay sincronizar UI.
- M6 `lib/api.ts` sin retry logic ni timeout. Network flaky (3G, mobile) → request fail directo.
- M7 Metadata solo en root layout, no per-page. SEO ruinoso.
- M8 globals.css con `.card`, `.btn-primary`, `.input`, `.tbl` clases custom en vez de `@apply` Tailwind. Doble mantenimiento.
- M9 No hay `next/image` (no hay imágenes hoy pero patrón debe estar).
- M10 Accessibility débil: tables sin `<caption>`, inputs sin `<label htmlFor>`, badges sin aria.

### Tests
- M11 Tests usan Supabase REAL como test DB. Lentos (5-10s/test), polluyen prod, bloquean parallel runs.
- M12 No hay `conftest.py` con fixtures reutilizables. Cada test repite `_get_tenant_id()`.
- M13 `test_conversiones.py::test_conversion_duplicada_409` order-dependent. Si pytest-random reordena, falla.
- M14 Sin coverage tracking (`pytest-cov`). No sabemos qué % del código se prueba.
